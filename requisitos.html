Requisitos Funcionais

RF.N Nome do Requisito Descrição
RF.1 Cadastro de Estacionamentos O sistema deve permitir o cadastro de diferentes estacionamentos, incluindo informações como localização, capacidade de vagas e horários de funcionamento.
RF.2 Visualização de Vagas Disponíveis Os usuários devem poder visualizar em tempo real as vagas disponíveis em cada estacionamento cadastrado.
RF.3 Reservar Vaga Os usuários devem ter a capacidade de reservar uma vaga de estacionamento com antecedência, quando disponível, através do aplicativo.
RF.4 Cancelar Reserva Os Usuários devem ter uma maneira de cancelar suas vagas para caso de adversidades ou se mudarem de ideia, desse modo o sistema irá atualizar para que outra pessoa possa pegar essa vaga.
RF.5 Registrar Entradas e Saídas Os usuários devem registrar quando entraram e quando saíram dos estacionamentos, para o sistema reconhecer que a vaga está sendo ocupada como deveria, já que, sem essa etapa, vagas poderão ser reservadas para o dia inteiro e não serão preenchidas, a maneira como esse check-in será feito está em debate.
RF.6 Indicação de Rotas O aplicativo deve fornecer rotas otimizadas para chegar ao estacionamento selecionado, considerando o tráfego em tempo real.
RF.7 Notificação em Tempo Real Os usuários devem receber notificações sobre o status de sua reserva, bem como alertas sobre vagas disponíveis ou eventos importantes, como mudanças nos horários de funcionamento.
RF.8 Feedback dos Usuários Deve ser possível para os usuários fornecer feedback sobre a experiência de estacionamento, ajudando a melhorar o serviço.
RF.9 Manter Estacionamento Os administradores devem ter acesso a um painel de controle onde possam gerenciar informações sobre os estacionamentos, verificar estatísticas de uso e responder a feedback dos usuários.
RF.10 Recomendação de Vagas Deve existir um sistema de recomendação de vagas personalizado, baseado nas preferências e histórico de utilização de cada usuário.
RF.11 Disponibilidade de Vagas Implementar um sistema de avaliação em tempo real da disponibilidade de vagas, considerando fatores como o tamanho do veículo e a acessibilidade da vaga.
RF.12 Gerenciamento de Conflitos Desenvolver um mecanismo para resolver conflitos de reservas, como quando dois usuários reservam a mesma vaga no mesmo horário, priorizando critérios como fidelidade do usuário ou urgência da reserva.
RF.13 Status de Vaga As Vagas deverão ter uma tag de status que indicará o quão prioritária ela será, se a vaga for específica para PCD, ela não poderá ser reservada por um usuário sem tal permissão, se a vaga for prioridade para um convidado do campus ou da empresa, ela não poderá ser reservada por um usuário que não o convidado, além disso as vagas em sua maioria terão status livre, a não ser que estejam ocupadas ou reservadas.

Requisitos não funcionais

NF.N Nome do requisito Descrição
NF.1 Desempenho O sistema deve ser capaz de lidar com um grande volume de usuários simultâneos sem degradação significativa do desempenho.
NF.2 Segurança O aplicativo deve garantir a segurança dos dados dos usuários, utilizando medidas como criptografia de dados e autenticação de dois fatores.
NF.3 Escalabilidade A arquitetura do sistema deve ser projetada para permitir escalabilidade fácil, para que possa lidar com o crescimento do número de usuários e estacionamentos cadastrados.
NF.4 Usabilidade A interface do aplicativo deve ser intuitiva e fácil de usar, mesmo para usuários inexperientes.
NF.5 Acessibilidade O sistema deve ser compatível com tecnologias assistivas, como leitores de tela e dispositivos de entrada alternativos, garantindo que os usuários com deficiências visuais ou motoras tenham acesso completo e eficaz ao aplicativo.
NF.6 Integração com Serviços Externos O sistema deve ser capaz de se integrar facilmente com serviços externos, como sistemas de pagamento online e APIs de tráfego em tempo real.
